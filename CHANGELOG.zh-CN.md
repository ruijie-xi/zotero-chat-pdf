# 更新日志

[English](CHANGELOG.md)

## 尚未发布

## 0.8.7

### 修复

- 将 ChatPDF 拉伸与 Zotero 内部面板隔离，为 XUL 布局显式分配完整 Zotero 内容根节点的宽度，并恢复明显损坏的原生上下文面板持久化宽度。

## 0.8.6

### 修复

- 使用基于完整宿主布局的显式面板拉伸替换仅调整最近兄弟元素的 XUL splitter，并增加可靠的宽度持久化、窗口边界处理、键盘拉伸及 Zotero 临时布局覆盖清理。

## 0.8.5

### 修复

- 移除侧栏最多占窗口 50% 的宽度限制，使 ChatPDF 面板可以在 Zotero 的可用工作区内继续拉宽。
- 恢复原生光标导航，并避免异步回答或“转换并发送”流程结束时抢走其他编辑框的焦点。

## 0.8.4

### 新功能

- 新增只读 Zotero 标注 Agent 工具，可列出标注，或搜索高亮正文、批注、标签及对应论文元数据。

### 修复

- 发布前校验版本号、XPI 元数据、更新 URL、兼容范围和 SHA-512 哈希；固定更新清单 Release 不再覆盖面向用户的 Latest Release。

### 安全

- 将存在漏洞的传递依赖 `adm-zip` 覆盖为 0.6.0；`npm audit --audit-level=low` 当前无已知漏洞。

## 0.8.3

### 修复

- 通过 `nsIDNSAddrRecord` 读取 Firefox DNS 回调结果，恢复 Zotero 7 中的 `web_search` 和 `web_fetch`，同时保留私有网络校验。

### 安全

- 使用 DOM 标签/属性白名单替换仅依赖正则表达式的 Markdown 清洗，并为危险元素、事件属性、URL 协议、SVG 和特权本地图片增加回归测试。
- 网络工具统一通过安全 HTTP 客户端：阻止本机、私网和保留地址，重新校验重定向，检查 DNS/MIME，并执行超时和流式响应上限。
- LLM 调试日志默认只记录元数据，提供显式的关闭/完整模式和保留期清理。
- 通过审查后的 package override 更新存在漏洞的传递依赖；生产与开发依赖审计目前均无已知漏洞。

### 变更

- 引入文库限定的 `libraryID:key` 来源身份、缓存键和 Zotero 工具参数，同时保留旧缓存读取能力。
- 输入框 mention 定义逐轮来源范围；没有 mention 时使用会话全部来源，只有本轮范围内尚未完成的转换会阻止发送。
- 请求取消贯穿 LLM 客户端、工具、网络请求、MinerU 轮询/下载/解压、会话修改和 UI 回调。
- 只读工具批次可以并行；任何包含修改操作的批次都按模型顺序串行。
- 当前工具输出保持完整并显示大小/token 估算；旧工具正文进入后续提示词时压缩为来源记录。
- 新增可配置的 `contextMaxChars` 预算，超限时在调用供应商前明确失败。
- 用每个 Zotero 窗口独立状态替换模块级全局面板状态，并确定性清理编辑器、监听器和流。
- 会话、历史索引、文档、分块和 manifest 使用串行原子写；缺失/损坏索引可重建，迟到的后台保存不能复活已删除会话。
- 持久化失败和取消的助手终态，并支持保存清空或只有来源的会话。
- 新增 MinerU 语言和超时配置，并提供准确的阶段错误。
- 使用容器查询、可换行控件和可滚动的展开工具结果改进窄面板体验。

### 开发体验

- 新增 TypeScript、ESLint、Vitest、CI 和统一的 `npm run verify` 命令。
- 增加来源身份/TurnScope、安全网络访问、Markdown 清洗、会话恢复和上下文预算的回归测试。
- 将 README 重写为简洁的用户指南，并让贡献者指南、LLM 工作流和更新日志保持中英文对应版本。
- 增加真实 Zotero 截图，并说明项目的个人使用定位和借助 AI 调整的路径。
- 删除过时的 Claude 专属文件、路线图/复审产物、未使用的旧代码与导出、本地化脚手架和模板图标。

## 0.8.0

### 新功能

- 新增 Zotero 文库代理工具：搜索条目、查看元数据、列出分类、读取当前选择，以及在聊天中添加/转换相关 PDF。
- 新增可点击的来源卡片，使聊天来源和消息来源快照能够打开对应的 Zotero PDF。
- 当 Zotero fetch 和 Zotero.HTTP 无法访问 MinerU CDN 时，为 MinerU 结果 ZIP 下载增加 Windows `curl.exe` 回退路径。

### 变更

- 给予代理更多自主权来添加和转换相关 Zotero PDF，同时把大范围转换警告保留为指导，而不是硬性拦截。
- 改进代理实时渲染，避免流式助手文本在工具调用块前后重复。
- Zotero 文库和分类匹配改为更宽松的、不区分大小写的部分匹配。

## 0.7.0

### 修复

- 将 MinerU 结果 ZIP 中的资源保存在每份已转换 PDF 旁边，使 Markdown 图片链接能找到对应的本地文件。
- 当 Zotero fetch 无法下载已完成的 MinerU 结果 ZIP 时，回退到 Zotero.HTTP。
- 写入前规范化 MinerU ZIP 资源路径，避免 Windows 版 Zotero 拒绝带斜杠的相对路径。
- 新转换结果保存到 `documents/<attachment-key>/document.md`，同时继续支持读取旧的根目录 Markdown 缓存。
- 针对上传 URL 请求、PDF 上传、结果轮询和结果 ZIP 下载，增加分阶段的 MinerU 网络错误。
- 为 API 主机、OSS 上传主机和结果 CDN 主机增加 MinerU 主机权限。

### 变更

- 将转换文档清单升级到版本 2，使缺少资源的旧分块缓存重新转换。
- 更新 README，以说明当前仅代理模式的工作流、长文档工具、MinerU 资源缓存布局和 CDN 故障排查。

## 0.6.0

### 新功能

- 使用 MinerU 按页码范围分块转换长 PDF，显示进度并复用已缓存的分块，使失败转换可以续跑而不用从头开始。
- 新增长文档聊天工具，用于列出分块、读取分块和搜索文档，使代理可以浏览书籍和大型已转换 PDF，而无需一次载入整份文档。

### 变更

- MinerU 轮询允许分块任务运行更长时间，并采用更小的页码范围，以提高书籍长度 PDF 的转换可靠性。
- 代理提示词优先使用搜索和分块读取来处理长文档。

## 0.3.0

### 新功能

- 持久侧边面板，始终显示在 Zotero 窗口右侧。
- 后台流持久化，因此切换条目或浏览历史时，LLM 生成仍可继续。
- 为正在进行的 MinerU 转换增加停止按钮。
- 为用户和助手消息增加时间戳。
- 更精确的历史时间戳。
- 新聊天首次回答后，由 LLM 生成会话标题。
- 历史列表中的会话标题可以编辑。
- 在用户消息下显示逐消息来源快照。
- 支持从阅读器标签拖放来添加来源。

### 修复

- 生成中途停止时，在会话历史中保留助手的部分回答。

## 0.2.0

### 新功能

- 首选项面板中可编辑系统提示词，并提供中英文重置默认值。
- 编辑并重新发送用户消息。
- 以原始 Markdown 复制助手消息。
- 为兼容的思考模型提供可折叠的推理/思考块。
- 在来源卡片上显示文档大小。
- 在来源卡片下显示总上下文用量摘要。
- 来源区域可调整大小。
- 阻止在来源尚待处理或正在转换时发送消息。
- 使用 KaTeX 渲染 LaTeX 数学分隔符。

### 变更

- 将首选项 `maxContextChars` 重命名为 `maxDocumentChars`，并把文档内容预算默认值提高到 300,000 字符。
- 在 LLM 流程各处增加 `[ChatPDF]` 调试日志。

### 修复

- 在添加当前用户消息前构建消息，修复 LLM 请求中用户消息重复的问题。
- 编辑后重新发送时，从会话状态重新渲染，修复陈旧 DOM。
- 反复尝试直到窗格元素可用，修复首选项面板初始化时序问题。
- 为大型文档增加更清晰的截断反馈。
